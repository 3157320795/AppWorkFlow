# 应用功能设计器

## 任务目标
- 本技能用于：根据移动应用产品名称，设计单机工具类应用的完整功能方案
- 能力包含：应用类型识别、功能模块规划、应用页面设计
- 触发条件：用户提供移动应用产品名称，需要设计该应用的功能方案

## 输出内容
- **产品主要功能**：功能模块清单及核心功能描述
- **应用页面设计**：一级主页面的详细布局设计（包含页面结构、区域划分、功能入口）

## 前置准备
- 确认产品名称：向用户确认或获取移动应用的产品名称
- 了解应用定位：确认是否为单机工具类应用（无联机功能、本地数据存储）
- 明确设计约束：参考 [references/design-framework.md](references/design-framework.md) 中的约束清单

## 操作步骤

### 步骤1：应用类型识别
根据产品名称识别应用类型，分为三类：

**实体型应用**：产品名称包含明确的实体关键词
- 实体关键词：猪、牛、羊、宠物、车、房、茶馆、店铺、仓库等
- 示例："猪猪来财" → 实体"猪猪" → 猪养殖管理系统
- 示例："好运茶馆" → 实体"茶馆" → 茶馆管理系统

**关系型应用**：产品名称包含明确的关系/动作关键词
- 关系关键词：养殖、租赁、销售、管理、服务、运营等
- 示例："车辆租赁" → 关系"租赁" → 车辆租赁管理系统
- 示例："员工管理" → 关系"管理" → 员工管理系统

**通用型应用**：产品名称不包含明确的实体或关系关键词
- 示例："读书笔记" → 通用工具类 → 阅读记录管理
- 示例："健身打卡" → 通用工具类 → 健身记录管理

**参考**：详见 [references/design-framework.md](references/design-framework.md) 中的应用类型识别规则

### 步骤2：功能模块设计
根据应用类型设计功能模块：

**实体型应用**：
- 功能模块数量：6-8个模块
- 模块命名规则：围绕实体展开
  - 实体核心模块：`{实体}档案`、`{实体}订单`、`{实体}饲养/维护`、`{实体}居所/位置`
  - 配套管理模块：`{场景}客户`、`{场景}员工`、`{场景}设施`、`{场景}异常`

**关系型应用**：
- 功能模块数量：6-8个模块
- 模块命名规则：围绕关系/动作展开
  - 核心业务模块：根据关系设计（如租赁→租赁管理、归还管理）
  - 支撑模块：客户管理、员工管理、统计管理、异常管理

**通用型应用**：
- 功能模块数量：2-4个模块
- 功能点总数：6-8个功能点
- 模块命名规则：围绕核心场景展开

**参考**：详见 [references/design-framework.md](references/design-framework.md) 中的功能模块设计规则

### 步骤3：应用页面设计
设计一级主页面的详细布局：

**页面数量**：3-5个（通常为4个）

**页面类型**：
1. **首页/主场景页面**：核心数据展示、事物介绍、功能入口、工具区域
2. **实体/物品页面**：分类展示、推荐内容、百科资讯、功能入口
3. **统计页面**：时间筛选、数据图表、详细列表
4. **我的页面**：用户信息、系统服务、账号管理

**页面结构**：从上到下划分为多个区域
- 数据展示区：展示关键数据指标
- 内容展示区：展示资讯、介绍、推荐
- 功能入口区：快捷功能入口或功能总览
- 工具区：单机小工具集合
- 个人中心区：用户信息、系统设置

**参考**：详见 [references/design-framework.md](references/design-framework.md) 中的应用页面设计规则

### 步骤4：合规性检查

**检查清单**：
- [ ] 是否为单机工具类应用（无联机功能）
- [ ] 是否严禁所有联网功能（扫码、社区、评论、分享、同步、在线查询等）
- [ ] 数据是否为本地存储
- [ ] 是否避免了金融、理财、股票相关内容
- [ ] 是否避免了单一资讯、知识查询、科普阅读类设计
- [ ] 是否避免了游戏属性
- [ ] 功能是否与产品名称相符
- [ ] 实体型/关系型应用：功能模块数量是否在6-8个之间
- [ ] 通用型应用：功能点数量是否在6-8个之间

### 步骤5：输出功能设计文档
- 使用 [assets/function-design-template.md](assets/function-design-template.md) 模板
- 输出产品主要功能和应用页面设计
- **输出内容必须包含**：
  - 产品主要功能：功能模块清单及核心功能描述
  - 应用页面设计：一级主页面的详细布局设计

## 使用示例

**示例1：实体型应用"好运茶馆"**
- 步骤1：识别实体"茶馆"，判定为实体型应用
- 步骤2：设计功能模块（茶馆茶室、茶馆员工、茶馆会员、茶馆经营、茶馆工具、茶馆异常，共6个模块）
- 步骤3：设计应用页面（茶馆、茶叶、统计、我的，共4个一级主页面）
- 步骤4：合规性检查通过
- 步骤5：输出产品主要功能和应用页面设计

**示例2：实体型应用"猪猪来财"**
- 步骤1：识别实体"猪猪"，判定为实体型应用
- 步骤2：设计功能模块（猪猪档案、猪猪订单、猪猪饲养、猪猪居所、猪场客户、猪场员工、猪场设施、猪场异常，共8个模块）
- 步骤3：设计应用页面（首页、猪只、统计、我的，共4个一级主页面）
- 步骤4：合规性检查通过
- 步骤5：输出产品主要功能和应用页面设计

**示例3：关系型应用"车辆租赁"**
- 步骤1：识别关系"租赁"，判定为关系型应用
- 步骤2：设计功能模块（车辆档案、租赁管理、归还管理、客户管理、员工管理、统计管理、异常管理，共7个模块）
- 步骤3：设计应用页面（首页、车辆、统计、我的，共4个一级主页面）
- 步骤4：合规性检查通过
- 步骤5：输出产品主要功能和应用页面设计

**示例4：通用型应用"读书笔记"**
- 步骤1：无实体或关系关键词，判定为通用型应用
- 步骤2：设计功能模块（书籍管理、阅读记录、笔记整理，共3个模块，6个功能点）
- 步骤3：设计应用页面（书架、笔记、统计、我的，共4个一级主页面）
- 步骤4：合规性检查通过
- 步骤5：输出产品主要功能和应用页面设计

## 应用类型识别规则

### 识别流程
1. 提取产品名称中的关键词
2. 判断是否包含实体关键词 → 实体型应用
3. 判断是否包含关系关键词 → 关系型应用
4. 均不包含 → 通用型应用

### 实体关键词清单

#### 动物类
猪、牛、羊、鸡、鸭、鱼、宠物、猫、狗、马、兔、鹅等

#### 物品类
车、房、设备、商品、库存、货物、车辆、汽车、房产等

#### 人物类
员工、客户、会员、学生、病人、会员、用户等

#### 场所类
茶馆、店铺、仓库、工地、农场、养殖场、餐厅、酒店、门店等

#### 产品类
茶叶、药品、食品、服装、书籍、商品等

### 关系关键词清单

#### 管理类
管理、运营、管家、系统等

#### 交易类
销售、租赁、买卖、交易等

#### 服务类
服务、配送、维修、保养等

#### 生产类
养殖、种植、生产、制造等

### 应用类型判断示例

| 产品名称 | 识别关键词 | 应用类型 | 系统定位 |
|---------|----------|---------|---------|
| 好运茶馆 | 茶馆（实体） | 实体型 | 茶馆管理系统 |
| 猪猪来财 | 猪猪（实体） | 实体型 | 猪养殖管理系统 |
| 宠物管家 | 宠物（实体）+管家（关系） | 实体型 | 宠物管理系统 |
| 车辆租赁 | 租赁（关系） | 关系型 | 车辆租赁管理系统 |
| 员工管理 | 员工（实体）+管理（关系） | 实体型 | 员工管理系统 |
| 读书笔记 | 无 | 通用型 | 阅读记录管理工具 |
| 健身打卡 | 无 | 通用型 | 健身记录管理工具 |

## 约束清单

### 必须遵守的约束
- [x] 单机工具类应用（无联机功能）
- [x] 数据仅为本地存储
- [x] 功能与产品名称相符
- [x] **实体型/关系型应用：功能模块数量6-8个**
- [x] **通用型应用：功能点总数6-8个**

### 必须避免的内容

#### 所有联网功能（严禁）
- 禁止：扫码识别、二维码扫描、条码扫描
- 禁止：社区功能、评论功能、点赞、关注
- 禁止：分享功能、社交分享、邀请好友
- 禁止：云同步、数据备份到云端、跨设备同步
- 禁止：在线查询、在线搜索、联网获取数据
- 禁止：推送通知（需要服务器的通知）
- 禁止：账号登录、第三方登录、注册功能
- 禁止：在线支付、内购功能
- 允许：本地提醒（基于设备本地的定时提醒）

#### 金融相关内容
- 禁止：股票查询、基金管理、投资理财、贷款计算等
- 允许：消费记录、预算管理（但不涉及投资理财）

#### 单一资讯类应用
- 禁止：纯新闻资讯、百科查询、科普文章阅读等
- 允许：结合记录、管理的知识库应用

#### 游戏属性
- 禁止：游戏玩法、游戏道具、游戏关卡、积分排名等
- 允许：游戏化元素（如打卡徽章、成就系统，但非核心玩法）

## 功能模块设计规则

### 实体型应用功能模块设计

#### 模块数量
**6-8个功能模块**

#### 模块命名规则

**实体核心模块**（围绕实体生命周期）：
| 模块名称 | 功能定位 | 示例 |
|---------|---------|------|
| {实体}档案 | 实体基本信息管理 | 猪猪档案、茶馆茶室、车辆档案 |
| {实体}订单 | 实体相关的订单/交易管理 | 猪猪订单、茶叶订单 |
| {实体}饲养/维护 | 实体的日常养护管理 | 猪猪饲养、车辆维护 |
| {实体}居所/位置 | 实体的位置或住所管理 | 猪猪居所、茶叶仓库 |

**配套管理模块**（支撑实体管理）：
| 模块名称 | 功能定位 | 示例 |
|---------|---------|------|
| {场景}客户 | 客户信息管理 | 猪场客户、茶馆会员 |
| {场景}员工 | 员工信息管理 | 猪场员工、茶馆员工 |
| {场景}设施 | 设施设备管理 | 猪场设施、茶馆设备 |
| {场景}异常 | 异常情况记录 | 猪场异常、茶馆异常 |

### 通用型应用功能模块设计

#### 模块数量
- **功能模块数量：2-4个**
- **功能点总数：6-8个**

#### 模块设计原则
1. 围绕核心场景展开
2. 每个模块包含2-4个功能点
3. 功能点总数控制在6-8个

#### 典型通用型应用模块模板

**案例：读书笔记**
| 模块名称 | 功能点 |
|---------|-------|
| 书籍管理 | 添加书籍、书籍分类 |
| 阅读记录 | 阅读打卡、进度追踪 |
| 笔记整理 | 创建笔记、笔记搜索 |

**案例：健身打卡**
| 模块名称 | 功能点 |
|---------|-------|
| 训练计划 | 创建计划、计划展示 |
| 打卡记录 | 每日打卡、历史查看 |
| 数据统计 | 训练统计、趋势展示 |

## 应用页面设计规则

### 页面数量
**一级主页面数量：3-5个（通常为4个）**

### 页面类型

#### 页面1：首页/主场景页面
**适用场景**：实体型应用、关系型应用

**页面结构**：
```
【首页/XX页面】
┌─────────────────────────────┐
│ 上方：核心数据展示区          │
│ - 数据项1：XX数量            │
│ - 数据项2：XX数量            │
├─────────────────────────────┤
│ 中间1：事物介绍区域           │
│ - 介绍入口1                  │
│ - 介绍入口2                  │
│ - 介绍入口3                  │
│ - 介绍入口4                  │
├─────────────────────────────┤
│ 中间2：常用功能快捷入口        │
│ - 功能1  功能2  功能3        │
├─────────────────────────────┤
│ 下方：休闲工具区域            │
│ - 工具1：XX工具              │
│ - 工具2：XX工具              │
│ - 工具3：XX工具              │
│ - 工具4：XX工具              │
├─────────────────────────────┤
│ 底部：功能总览入口            │
│ - 入口1  入口2  入口3 ...    │
└─────────────────────────────┘
```

**区域说明**：
- 数据展示区：展示关键数据指标（如会员数量、员工数量）
- 事物介绍区域：相关事物介绍入口（如饮茶礼仪、饮茶工具等）
- 常用功能入口：高频使用的功能快捷入口
- 休闲工具区域：2-4个单机小工具
- 功能总览入口：所有功能模块的入口

**适用场景**：展示实体信息、物品分类

```
│ 上方：分类展示区              │
│ - 分类1  分类2  分类3        │
├─────────────────────────────┤
│ 中间1：推荐内容区             │
│ - 推荐1：XX推荐              │
│ - 推荐2：XX推荐              │
├─────────────────────────────┤
│ 中间2：动态通知/百科区        │
│ - 内容1：XX百科              │
│ - 内容2：XX通知              │
├─────────────────────────────┤
│ 下方：相关功能入口            │
│ - 入口1  入口2  入口3 ...    │
└─────────────────────────────┘
```

**区域说明**：
- 分类展示区：实体/物品的分类
- 推荐内容区：推荐的实体/物品
- 动态通知/百科区：相关知识、资讯
- 相关功能入口：与实体/物品相关的功能入口

### 单机小工具设计

#### 工具类型
| 类型 | 说明 | 示例 |
|-----|------|------|
| 计时工具 | 倒计时、计时器、时钟 | 泡茶时钟、烹饪计时器 |
| 计算工具 | 单位转换、数值计算 | 控温大师、茶量挑战 |
| 记录工具 | 快速记录、模板记录 | 快速记账、快速日志 |
| 查询工具 | 本地数据查询 | 价格查询、库存查询 |

#### 工具设计要点
- 工具数量：每个页面2-4个小工具
- 工具功能：纯本地计算，不依赖网络
- 工具命名：简洁明了，如"XX时钟"、"XX大师"、"XX挑战"
- 交互反馈：用户输入后提供明确的反馈信息

### 页面设计示例

#### 案例：好运茶馆 - 茶馆页面
```
【茶馆页面】

上方区域：茶馆数据展示
- 会员数量：XX人
- 员工数量：XX人

中间区域1：茶馆事物区域（4个事物介绍入口）
- 饮茶礼仪介绍
- 饮茶工具介绍
- 茶叶介绍
- 泡茶技巧

中间区域2：常用功能快捷入口
- 会员管理
- 员工管理
- 茶室管理

下方区域：茶馆休闲工具（4个单机小工具）

- 茶叶板画：涂鸦工具

底部区域：功能总览入口（6个功能入口）
- 茶馆茶室、茶馆员工、茶馆会员、茶馆经营、茶馆工具、茶馆异常
```
